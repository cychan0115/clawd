## 1. 项目概述

开发一个 Web 页面，用于**实时监控和展示社区队列状态**，包括：
1. **队列总览面板** - 显示当前社区有多少个队列，以及各队列状态统计
2. **报告打印队列** - 专门展示报告打印相关的队列任务列表

### 1.1 目标用户
- 社区管理员
- 运营人员
- 技术支持

### 1.2 核心价值
- 一眼看清社区队列负载情况
- 快速发现异常队列（堆积、阻塞）
- 追踪报告打印任务的执行状态

## 2. 功能需求

### 2.1 页面一：队列总览面板（Dashboard）

#### 2.1.1 顶部统计卡片
页面顶部显示 4 个关键指标卡片，横向排列：

| 指标 | 说明 | 数据示例 |
|------|------|---------|
| **总队列数** | 当前社区活跃队列总数 | 12 |
| **运行中队列** | 正在处理任务的队列数 | 8 |
| **空闲队列** | 无任务在处理的队列数 | 3 |
| **异常队列** | 堆积/阻塞/报错的队列数 | 1 |

- 卡片需要有视觉区分：运行中（绿色）、空闲（灰色）、异常（红色/橙色）
- 数字要大且醒目，建议用 36px+ 字体
- 卡片点击后可下钻到对应队列列表

#### 2.1.2 队列状态分布图
一个可视化区域，展示队列状态的分布：

**方案 A（推荐）**：饼图 / 环形图
- 运行中（绿色）
- 空闲（蓝色）
- 阻塞（橙色）
- 错误（红色）

**方案 B**：横向柱状图
- X 轴：状态
- Y 轴：队列数量

#### 2.1.3 队列列表详情
一个表格展示所有队列的详细信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 队列名称 | 队列的标识名 | report-queue, email-queue |
| 队列类型 | 队列的分类 | 打印/邮件/数据处理/...
| 当前状态 | 运行中/空闲/阻塞/错误 | 运行中 |
| 待处理任务数 | 堆积的任务数量 | 23 |
| 正在处理 | 当前正在执行的任务数 | 2 |
| 处理速度 | 最近1分钟处理的任务数 | 5/min |
| 最后活动 | 最后处理任务的时间 | 2026-06-12 14:55:00 |
| 操作 | 查看详情/清空队列/重试 | [按钮] |

**表格功能：**
- 支持按状态筛选（全部/运行中/空闲/阻塞/错误）
- 支持按队列名称搜索
- 支持按待处理任务数排序（发现堆积）
- 行内状态用颜色标签展示（绿/灰/橙/红）
- 待处理任务数 > 50 时高亮显示
- 支持分页，每页默认 20 条

#### 2.1.4 实时更新
- 数据每 30 秒自动刷新一次
- 右上角显示 "最后更新时间：XX:XX:XX"
- 提供手动刷新按钮
- 数据变化时卡片数字有动画过渡效果

### 2.2 页面二：报告打印队列（Report Print Queue）

#### 2.2.1 页面用途
专门展示和管理与"报告打印"相关的队列任务。

#### 2.2.2 顶部筛选栏
- 时间范围筛选：今天/最近7天/最近30天/自定义
- 状态筛选：全部/待打印/打印中/打印完成/打印失败
- 报告类型筛选：全部/日报/周报/月报/自定义报告
- 搜索框：按报告名称、用户ID、任务ID搜索

#### 2.2.3 统计摘要
- 今日打印请求总数
- 打印成功数 / 失败数
- 平均打印等待时间
- 当前排队中的报告数量

#### 2.2.4 报告打印任务列表
表格字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| 任务ID | 唯一标识 | RPT-20260612-001 |
| 报告名称 | 用户可见的报告标题 | 社区月度活跃度报告 |
| 报告类型 | 分类 | 月报 |
| 请求用户 | 发起打印的用户 | user_12345 |
| 提交时间 | 进入队列的时间 | 2026-06-12 14:30:00 |
| 开始时间 | 实际开始打印的时间 | 2026-06-12 14:35:00 |
| 完成时间 | 打印结束时间 | 2026-06-12 14:36:00 |
| 当前状态 | 待打印/打印中/成功/失败 | 打印成功 |
| 页数 | 报告页数 | 15 |
| 打印机 | 分配的打印机 | Printer-A1 |
| 错误信息 | 失败时显示原因 | 缺纸/连接超时/... |
| 操作 | 重试/取消/查看日志 | [按钮] |

**状态标签颜色：**
- 待打印：蓝色
- 打印中：橙色（可加旋转 loading 图标）
- 打印成功：绿色
- 打印失败：红色

#### 2.2.5 批量操作
- 勾选多行后，可批量：重试、取消、删除
- 顶部显示"已选择 X 项"的批量操作栏

#### 2.2.6 任务详情弹窗
点击任务行可展开或弹出详情：
- 完整的任务元数据
- 打印参数（纸张大小、彩色/黑白、份数）
- 执行日志（时间线形式）
- 错误堆栈（失败时）

## 3. 数据模型（API 接口需求）

### 3.1 后端需要提供的 API

```
GET /api/v1/queues/overview
# 返回队列总览统计数据

GET /api/v1/queues
# 返回队列列表（支持分页、筛选、排序）
Query: page, pageSize, status, search, sortBy, sortOrder

GET /api/v1/queues/{queueId}/tasks
# 返回指定队列中的任务列表

POST /api/v1/queues/{queueId}/clear
# 清空队列（危险操作，需确认）

POST /api/v1/queues/{queueId}/retry
# 重试失败任务

GET /api/v1/reports/queue
# 返回报告打印队列列表
Query: timeRange, status, reportType, search, page, pageSize

GET /api/v1/reports/queue/{taskId}
# 返回单个报告打印任务详情

POST /api/v1/reports/queue/{taskId}/retry
# 重试打印任务

POST /api/v1/reports/queue/{taskId}/cancel
# 取消打印任务
```

## 5. 技术栈建议

### 5.1 前端（推荐）
- **框架**：Vue 3 + TypeScript 或 React + TypeScript
- **UI 组件库**：Element Plus (Vue) / Ant Design (React)
- **图表库**：ECharts 或 Chart.js
- **状态管理**：Pinia (Vue) / Zustand (React)
- **HTTP 请求**：Axios
- **实时更新**：setInterval 轮询（30秒）或 WebSocket（进阶）

### 5.2 后端（如需开发）
- **语言**：Node.js / Python / Go
- **框架**：Express / FastAPI / Gin
- **数据库**：PostgreSQL / MySQL（存储历史数据）
- **缓存**：Redis（队列实时状态）
- **任务队列**：Bull / Celery / 自定义

### 5.3 部署
- 前端：静态文件托管（Nginx / CDN）
- 后端：Docker 容器化部署
- 环境：开发 / 测试 / 生产

## 6. 交互与体验细节

### 6.1 加载状态
- 页面首次加载：骨架屏（Skeleton）
- 数据刷新时：表格顶部显示 "更新中..." 轻提示
- 操作提交时：按钮显示 loading，防止重复提交

### 6.2 空状态
- 无队列时：友好提示 + 引导操作图标
- 无打印任务时：提示 "暂无打印任务"

### 6.3 错误处理
- API 请求失败：Toast 提示 + 重试按钮
- 数据解析错误：降级显示 + 错误日志入口
- 危险操作（清空队列）：二次确认弹窗

### 6.4 操作反馈
- 重试成功：绿色 Toast "任务已重试"
- 取消成功：灰色 Toast "任务已取消"
- 操作失败：红色 Toast + 错误详情

## 7. 权限与安全

### 7.1 权限控制
- 页面需要登录访问
- 角色区分：
  - 管理员：可查看、可操作（重试/清空/取消）
  - 观察员：仅可查看

### 7.2 安全要求
- 所有 API 请求需携带 JWT Token
- 危险操作（清空队列）需二次确认
- 操作日志记录（谁、何时、做了什么）
- 支持操作审计追溯

---

## 8. 非功能需求

### 8.1 性能
- 页面首屏加载 < 2 秒
- API 响应时间 < 500ms
- 表格支持虚拟滚动（数据量大时）
- 前端数据缓存，减少重复请求

### 8.2 兼容性
- 浏览器：Chrome 90+ / Firefox 88+ / Safari 14+ / Edge 90+
- 不支持 IE

### 8.3 可扩展性
- 预留其他队列类型入口（邮件队列、短信队列、数据同步队列）
- 图表类型可配置切换
- 支持多社区/多环境切换（未来）
